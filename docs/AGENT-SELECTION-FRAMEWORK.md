# Agent 选型框架

## ReAct 的本质

**思考 → 行动 → 观察 → 循环**

这个循环本身就是处理不确定性的最小单元。当你不知道答案时，你尝试、观察、调整。

## deepAgent 的本质

**ReAct + 可插拔的能力模块**

```
deepAgent = ReAct Core + Σ(Primitives)

其中 Primitives ∈ { memory, skills, files, subagents, plan, hitl, summary, ... }
```

这意味着：
- 最简单的 deepAgent 就是 ReAct
- 复杂的 deepAgent 是增强版 ReAct
- 没有断层，是连续的

## 完整的 Agent 选型框架

```
                    ┌─────────────────┐
                    │   你的问题域     │
                    └────────┬────────┘
                             │
              ┌──────────────┴──────────────┐
              ▼                              ▼
      路径确定吗？                      路径不确定？
      (流程已知)                        (需要探索)
              │                              │
              ▼                              ▼
     ┌────────────────┐            ┌────────────────┐
     │   LangGraph    │            │ ReAct/deepAgent │
     │   流程编排      │            │   自主探索      │
     └────────────────┘            └────────┬───────┘
                                            │
                                     复杂度如何？
                                            │
                        ┌───────────────────┼───────────────────┐
                        ▼                   ▼                   ▼
                      简单                 中等                复杂
                        │                   │                   │
                        ▼                   ▼                   ▼
                     ReAct            deepAgent            deepAgent
                   (或最简配置)        (部分原语)            (全部原语)
```

## 一点补充

Flow orchestration（LangGraph）在**确定性场景**仍有价值：
- 合规审计需要可预测路径
- 多步骤审批流程
- 状态机明确的业务流程

但核心观点是：**在不确定性这条轴上，ReAct → deepAgent 是完备的。**

这个认知简化了选型：遇到不确定问题，不用纠结，从 ReAct 开始，按需叠加原语，deepAgent 框架天然支持这种渐进增强。
